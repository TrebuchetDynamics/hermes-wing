# ADR 0016: Enroll Android with one-time pairing codes

Status: accepted
Date: 2026-07-13

Hermes Agent enrolls Android clients with short-lived, single-use pairing codes. A QR or `navivox://connect` payload identifies the normalized endpoint, pairing code, and requested scopes; after explicit operator review, Android exchanges the code over HTTPS for a scoped operator token and stores that token in platform secure storage.

## Consequences

- Bearer tokens and compatibility superuser keys never appear in URLs, QR payloads, Android shared text, logs, screenshots generated by the app, or clipboard flows.
- Hermes Agent owns code creation, expiry, single-use redemption, scope approval, token issuance, and audit metadata.
- Navivox shows the endpoint and requested scopes before redemption and fails closed on malformed, expired, reused, or origin-mismatched payloads.
- Existing Android connect-intent parsing is ingress only; Dart performs strict payload validation and the authenticated exchange.
- Manual scoped-token entry remains a developer fallback. Manual superuser-key enrollment requires an explicit warning and is not the default flow.
